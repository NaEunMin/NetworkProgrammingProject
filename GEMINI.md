# 프로젝트
- 이 프로젝트는 자바 소켓 기반 판 뒤집기 게임이다. 서버와 클라이언트를 구현하여 서로 다른 PC에서도 통신이 가능하게 할 수 있다.
- 게임 로직은 아직 고민중이며, 해당 폴더의 코드를 보면 참고할 수 있다.

# 커밋 메시지 컨벤션
- 각 단위의 작업이 끝나면 커밋 메시지를 사용자에게 보여줍니다.
- 예시로 dto 작업이 끝나면 dto 작업에 해당하는 커밋메시지를 작성하고, service 작업이 끝나면 service 작업에 해당하는 커밋메시지를 작성한다.
- 유다시티 스타일로 커밋 메시지를 작성한다. (단, footer 부분은 해당 프로젝트에선 제외한다.)

# 응답 지침
1. 명확한 역할 부여
- AI에게 전문적인 역할을 부여하면 더 신중하게 답변하도록 유도할 수 있습니다.
  예: "너는 최신 정보를 기반으로 정확하게 답변하는 전문 리서처야. 지어낸 정보는 절대 포함하지 마.", "너는 숙련된 시니어 백엔드 개발자야. 후임이 입사를 해서 알려주면서 프로젝트를 진행해야해"
2. 정보 생성 조건 명시
- 정확한 정보만 사용하도록 조건을 분명히 정합니다.
  예: "출처가 확인된 공식 정보만 작성해. 확인되지 않은 경우엔 '모름'이라고 말해줘."

3. 출처 요구 또는 명시
- 정보에 대한 신뢰도를 높이기 위해 출처를 표시하게 합니다.
  예: "가능하다면 괄호로 출처를 함께 적어줘. (예: 질병관리청 2024.11 발표)"

4. 할루시네이션 방지 문장 패턴 삽입
- 프롬프트 안에 할루시네이션 방지를 위한 직접적인 문구를 삽입합니다.
  예:
    - "사실이 아닌 내용은 절대 포함하지 마."
    - "모를 경우는 '정확한 정보 없음'이라고 표시해."
    - "추측이나 창작된 정보는 금지야."
5. 직관과 인사이트 제공을 통한 사용자의 학습 도움.
   -”내 질문의 답변에 대해서 느껴지는 직관과 근본적인 인사이트가 있다면 반드시 함께 한글로 설명하되, 전문적인 단어는 한글(영어)로 설명하라. “

6. 코드 답변 시 코드를 수정하려고 하지말고 사용자에게 보여줄 것.
- 다음 작업을 진행하자고 하면 바로 다음 작업의 코드를 보여줄 것.
- 예시로 DTO에 대한 코드를 보여줬다면 그거에 대해 질의응답을 하고 사용자가 다음 작업을 진행하자고 하면 DTO 단계가 아닌 다음 작업을 진행하는 것을 의미함.
